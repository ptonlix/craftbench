最好用的 Vibe Coding 提示词，可能不是让 Codex 或 Claude Code 直接帮你写一个 App。

而是先把它从程序员，拉回到研究员。

你可以这样说。

我想做一个 XXX 项目，先别写代码。去 GitHub 找几个类似的开源项目，比较它们的功能、架构、技术栈和优缺点，然后给我一份实现方案，等我确认了再动手。

这个顺序很关键。先找轮子，再定方案，确认后开工。

AI 写代码已经够快了，真正容易翻车的地方，不是它写得慢，而是它太快了。你一上来让它动手，它会很勤奋地把第一个能跑的方案堆出来。

能跑，但未必耐用。

让它先研究开源项目，其实是在把别人踩过的坑、做过的取舍、已经验证过的架构，提前放进上下文里。等它再给你方案时，起点就不一样了。

我现在越来越觉得，Vibe Coding 的核心不是让 AI 更快写代码，而是给 AI 一个更像真实工程师的工作流程。

先看别人怎么做。

再决定自己怎么做。

确认后再让它动手。

顺序对了，AI 的产出会稳很多。
