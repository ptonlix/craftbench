BeeWeave 写作风格学习与激活流程图

Layout: 16:9 左到右五阶段流程。每个阶段一个粗边框容器，阶段之间用箭头连接。第四阶段是醒目的人工确认闸门，未确认分支回到 pending，确认分支进入 active rules。

STEPS:
1. 人工改稿，展示一张带删除线和新增句子的文稿
2. Trace 记录，保存改动方向、路径和证据
3. Learner 提炼，生成 pending 候选卡片
4. Evolver 审阅，检查重叠、证据、适用范围和回滚方式
5. 用户确认，确认后进入 Active Rules，未确认则继续观察

CONNECTIONS:
- 主流程从左向右
- 用户确认位置画一个明显闸门和手掌确认图标
- 从 Active Rules 画一条细箭头回到下一次写作
- 从未确认分支画回环箭头返回 Pending

LABELS:
- 只显示：人工改稿、Trace、Learner、Pending、Evolver、证据审阅、用户确认、Active Rules、继续观察、下一次写作
- 底部主句只显示：风格学习不能偷偷发生
- 中文必须准确清晰，不增加长段文字或乱码

COLORS:
- 暖米色背景 #F5E6D0
- 暖橙色 #E8751A 表现改稿和学习动作
- 金黄色 #F4A623 表现候选与证据
- 深棕色 #4A2C20 表现文字、框架和 active rules
- 猩红色 #C0392B 强调用户确认闸门与未确认分支
- 色值仅用于渲染，不得显示在图中

STYLE:
- Screen print / silkscreen editorial process poster
- 粗几何容器、粗箭头、半色调网点、纸张颗粒和轻微套色偏移
- 平涂色块，最多五种颜色，无渐变
- 图标为简化剪影和符号，不写实、不使用 3D、无复杂 UI
- 中文粗体海报字，核心结论最大，其他文字保持短标签
- Clean composition with generous white space

ASPECT: 16:9, 2K, medium complexity.
