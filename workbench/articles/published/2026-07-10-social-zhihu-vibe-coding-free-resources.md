最近 Vibe Coding 的课遍地开花，价格从几百到几千不等。我翻了几门课的大纲和内容去看，结果发现一个挺荒谬的事实。它们拿出来当卖点的"独家方法论"，几乎全部来自 GitHub 上三个免费的开源仓库。

这事让我想明白了一件事。这些课为什么能卖这么贵？不是因为内容有多稀缺，而是抓住了两样东西。信息差，和 FOMO。很多人看到身边的人在用 AI 写代码做产品，心里慌，急着找捷径，结果花几千块买了一个别人打包好的 GitHub star 列表。

但话说回来，这三个仓库确实组成了 Vibe Coding 最扎实的一条自学路线。我按顺序理出来。

第一个是 Easy-Vibe，16K star，DataWhale 出品。定位是入门实战，纯项目驱动。入门做 AI 小游戏建立手感，进阶做 Web 前端原型，高阶搞桌面应用和多 Agent 协同。零基础也能跟。

第二个是 vibe-coding-cn，1.4K star，方法论核心。它回答了一个问题。为什么大多数人用 AI 编程总翻车？答案不是模型不够强，是你只给了 AI 一句需求，没给完整的工程上下文。这个仓库教的是标准化 Prompt 规范、个人 Skill 库沉淀、完整 Workflow 搭建。这一步是分水岭。

第三个是 awesome-vibe-coding，5.5K star，工具百科。浏览器工具、IDE、移动端、CLI、社区文档，全收录。前两个教思路和方法，这一个是兵器库。

学习顺序不能乱。Easy-Vibe 跑通第一个项目，vibe-coding-cn 学工作流告别翻车，awesome-vibe-coding 补工具链，再用真实项目收尾。

一周能跑通第一个 AI 小游戏，两周做出 Web 原型，一个月站稳工作流，三个月一个人顶小团队。

这三个仓库免费，开源，中文友好。从用 AI 写代码到用 AI 做产品，中间差的就是这条路线的距离。

那些几千块的课，卖的不是内容，是有人帮你把免费信息整理了一遍的排版费。你愿意花时间自己趟，它就是最好的老师。
